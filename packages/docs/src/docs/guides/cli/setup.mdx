Skaff reads configuration from three sources, merged in this order: environment
variables, `~/.config/skaff/settings.json`, and runtime flags. Use the CLI to
keep everything aligned.

## Manage settings

```bash
# add directories that contain template repositories
skaff config add ~/templates TEMPLATE_DIR_PATHS

# add a GitHub repository (defaults to the main branch)
skaff config add github:timonteutelink/example-templates TEMPLATE_DIR_PATHS

# optionally tell Skaff where to find existing projects
skaff config add ~/code PROJECT_SEARCH_PATHS

# change the package runner exposed to template commands
skaff config set NPM_PATH bunx

# inspect the current configuration
skaff config get --format table
```

Set `SKAFF_CONFIG_PATH` to a **directory** to relocate the config file
(`settings.json` will be created inside it).

## Cache and dev settings

- `SKAFF_CACHE_PATH` overrides the template cache directory. When unset, Skaff
  uses an OS temp directory.
- `--dev-templates` (or `SKAFF_DEV_TEMPLATES=1`) allows dirty template
  repositories and skips strict commit hash checks during template development.

## Load template repositories

Clone or refresh templates on demand:

```bash
skaff template load github:org/templates --branch main
skaff template list --format table
```

Use `skaff template reload` to fetch the latest commits for cached repos,
`skaff template load --refresh` to force a refresh for a specific repo, or
`skaff template erase-cache` to clear stale copies.
